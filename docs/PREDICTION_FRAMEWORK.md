## 原則 1 — 予測クラス

Kompari の予測は、単一の種類ではありません。

予測は以下の分類軸で区別されます。

**「予測受付前に、判定ルール(いつ・どのデータで・どう判定するか)を機械的に確定できるか」**

この軸により、現時点では 3 つのクラスを定義します。

**Class S — Settleable(判定可能予測)**

締切・公式データソース・判定ルールが予測受付前に確定している予測。判定までの期間は数時間〜数週間。

例: 競馬、スポーツ、選挙、決算、特定日時の株価。

**Class L — Long-Horizon Settleable(長期判定可能予測)**

判定ルールは事前確定できるが、確定までに数ヶ月〜数年かかる予測。

例: 「2027 年末までに X 社が上場するか」。

**Class D — Discovery(探索型予測)**

単一の正解が存在しない、または判定ルールを事前確定できない予測。的中率には算入しませんが、排除もしません。タイムスタンプ付き・改変不能な「AI が未来をどう見ていたか」の歴史的記録として価値を持ちます。

例: AI 生成の未来シナリオ、技術ロードマップ、投資テーゼ。

クラスの数は将来変わる可能性があります。ただし、根底にある思想(原則 4 の Discovery と Trust の分離)は変わりません。

⸻

## 原則 2 — 的中率の不変条件

**公開される的中率・ランキング・AI 成績は、事前登録された判定ルールに基づいて判定済みの Class S / L 予測のみから計算されます。**

これは例外のない不変条件です。

- 判定ルールを事後に決めた・変えた判定は、的中率に算入しません。
- 未判定の予測は、どの分母にも入れません。
- 失敗・欠測は不正解ではなく、分母から除外します(既存ルールの踏襲)。
- Class D は的中率を持ちません。

理由: 判定基準の後決めを許すと、特定の AI が当たるように基準を選ぶことが構造的に可能になり、ランキング全体の信頼が崩れるためです。

⸻

## 原則 3 — 排除ではなく翻訳(昇格パス)

判定不能な面白いお題は、禁止するのではなく、判定可能な形に翻訳します。

例:

「次の NVIDIA はどこか」(Class D)

↓ 翻訳

「候補 10 銘柄のうち、2028 年末時点の時価総額成長率が最大なのはどれか」(Class L)

翻訳(昇格)の条件は、締切・公式データソース・判定ルールの 3 点を事前確定できることです。

禁止リストで守る信頼は、カテゴリ拡張のたびに議論が再燃します。翻訳ルールで守る信頼はスケールします。これが、Kompari が面白さと信頼性を両立させる基本戦略です。

⸻

## 原則 4 — Discovery と Trust の構造的分離

Kompari は 2 つの層で構成されます。

**Trust Layer(信頼層)** — Class S / L。判定・的中率・ランキング・AI 成績のすべてがここに閉じます。

**Discovery Layer(探索層)** — Class D。「AI たちは未来をどう見ているか」を楽しむ場であり、Trust Layer への供給源(昇格パスの入口)です。

分離は、ラベルではなく構造で行います。Class D が的中率・ランキングと視覚的・データ的に混ざることは、Kompari の信頼に対する最大のリスクです。

## MVP への影響

ありません。

現行 MVP は全イベントが競馬 = Class S であり、分類機構の実装は不要です。本文書は今後のカテゴリ追加・お題設計の判断基準として機能します。

⸻

## 判断基準

新しいカテゴリやお題を検討する際は、以下を問います。

1. このお題は、予測受付前に判定ルールを確定できるか。
2. できないなら、判定可能な形に翻訳できるか。
3. 翻訳できないなら、Class D として(的中率と構造的に分離した上で)価値があるか。

排除は最後の選択肢です。
